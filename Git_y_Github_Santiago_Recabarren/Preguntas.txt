1- ¿Quién inventó el sistema de control de versión Git y por qué?
	Git es un proyecto de código abierto maduro y con un mantenimiento activo que desarrolló originalmente Linus Torvalds, el famoso creador del kernel del sistema operativo Linux, en 2005. Se creó pensando en la eficiencia, la confiabilidad y compatibilidad del mantenimiento de versiones de aplicaciones cuando estas tienen un gran número de archivos de código fuente.

2- ¿A quién pertenece actualmente Github y por qué?
	El 4 de junio de 2018, Microsoft compró GitHub por la cantidad de 7500 millones de dólares. La compra de GitHub es un paso más en la inclinación de Microsoft para enfocarse todo lo posible al desarrollo, haciendo una vez más hincapié en que el núcleo de la empresa lo forman los desarrolladores y nada más.


3- ¿Hay otra forma que no sea la terminal para trabajar con Github?
	Si, se puede trabajar con Github "manualmente" por decirlo de alguna manera, trabajando los archivos desde el navegador.
